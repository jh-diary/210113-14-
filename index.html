topic 11. 반응형 디자인
responsive web- 웹페이지의 요소들이 화면의 크기 변화에 반응해서 최적화된 모습으로 바뀌는 것
원하는 태그에 대하여 화면의 크기에 따라 디자인을 다르게 할 것이라면 디자인에 관한 css태그들이 사용될 것이다.
mediaquary 파일과 같이 css스타일 태그를 작성하고 반응형 디자인을 작성할 것이다.
반응형 디자인을 작성할 때 어느정도의 화면이 조정되면 mediaquary가 작동할 것인지 정해야하기때문에 검사 기능을 활용하여 px정도를 어느정도 가늠한다.
파일과 같이 본인이 800px보다 화면이 커질 때 화면을 조정하고싶다면 screen width > 800px 일 것이다.
이를 mediaquary로 바꾸려면 @media(min-width:800px){}와 같이 작성하면 된다.
두 문장은 같은 뜻을 지니고있다.
반대로 800px보다 화면이 작아질 때 화면을 조정하고싶다면 800px > screen width 일 것이다.
이는 mediaquary로 바꾸려면 @media(max-width:800px){}와 같이 작성하면 다음태그에 적는 내용이 적용될 것이다.

topic 13. css코드의 재사
css 코드를 적용하는 파일이 여러가지라고한다면 이 상태로 진행이된다고 가정했을 때 한가지의 조정이 굉장히 많은 일의 반복으로 진행될 것이다.
극단적으로 상황을 생각했을 때 이렇게 복사,붙여넣기 식의 코드 사용은 적합하지않. 한가지의 독자적인 css 코드를 만든다고 생각하면 예외이긴하다
그래서 따로 css 파일을 만들어서 style태그를 제외한 css코드를 옮겨놓고 link태그를 사용한다.
link태그 사용 후 rel="stylesheet" 이건 그대로 두고 href를 링크 주소를 걸어주면 코드가 다운되어 적용된다!
결과적으로 수정을 할 때에 style.css페이지만 수정을 한다면
캐싱이라는 개념은 저장하다라는 의미인 이는 link를 이용한 stylesheet를 다운받아 사용하게 되는 것이 실제로 트래픽을 더 사용하게 하는 것이라고 생각이 들지만 사실 캐싱덕분에 그렇지않고 더 효율적이다라는 것을 알 수 있게 한다.
캐싱은 한번 웹사이트가 열리면 거기에 필요한 정보들이 컴퓨터안에 저장이되었다가 같은 파일을 열어보게 되면 트래픽을 사용안하고 컴퓨터 내부의 저장되어있는 요소를 사용하기 때문에 사용자입장에서도 속도가 빨라지고 개발자입장에서도 트래픽 사용료가 줄어드니까 이익이되는 굉장히 유용한 기술이다.
